Recognize multiline URLs
Patch is available for version: alpine 2.00 alpine 2.01 alpine 2.10 alpine 2.11 alpine 2.20 alpine 2.21 alpine 2.22 alpine 2.23 alpine 2.24 alpine 2.25 alpine 2.26
 
Many times one receives URLs in messages that have been folded in several lines. When this happens, Alpine can only recognize the first line in the URL, but other lines are not.
 
This patch searches for a continuation of a URL in several lines. This is done automatically if the URL is delimited between < and >. This case should be handled without problems.
 
The option [X] Enable Recognition of Long URLS without Delimiter can be used to go even further. If a URL is not delimited between < and >, then this option can be used to search for continuation of those URLs. In this case, Alpine can make mistakes and find a string longer than the URL. It should be clear what Alpine selected as the URL from what you can see in the screen. If Alpine made a mistake, you can edit the URL before sending it to the browser.
